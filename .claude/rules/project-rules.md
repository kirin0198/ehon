## Project Overview

3〜5歳（未就学児）向けの、著作権フリーの童話・昔話を読める日本語の絵本 Web アプリ。
本棚 (2 バリアント) + 絵本ビュアー (2 バリアント) + Tweaks パネル (フォント・夜モード・ふりがな等) を提供する。
将来的には LLM (`claude-haiku-4-5` 等) によるオリジナル童話生成を統合予定。

現状の作業ディレクトリには Claude Design 由来のハイファイモック (React 18 + Babel Standalone, CDN 直読み込み)
が配置されている。本実装ではこれをデザインリファレンスとして、ビルド環境付きの React + Vite +
TypeScript プロジェクトに作り直すことを基本方針とする。

## Tech Stack

- Language: TypeScript 5.x
- Runtime: Node.js 20 LTS 以降
- Framework: React 18 + Vite
- Package manager: pnpm (推奨。npm でも可)
- Linter: ESLint (`eslint-config-react`, `@typescript-eslint`)
- Formatter: Prettier
- Test framework: Vitest + @testing-library/react
- E2E test (HAS_UI=true): Playwright
- Styling: CSS Custom Properties (デザイントークンはモックの `styles/ehon.css` を踏襲)

> モック踏襲方針:
> - モック上の `Story` 型 (data/stories.js) は TypeScript 型定義として共有する。
> - CSS 変数 (`--paper`, `--ink`, `--terracotta` 他) はそのまま `:root` レイヤに移植する。
> - ふりがな記法 `桃太郎{ももたろう}` のパーサ (`renderRuby`) は `src/lib/ruby.ts` 等に再実装する。
> - Tweaks パネルランタイム (`tweaks-panel.jsx`) は本実装では使用しない。設定ストアは Zustand
>   または React Context + localStorage に置き換える。

## Coding Conventions

### Naming Rules

| Target               | Rule              | Example |
|----------------------|-------------------|---------|
| Variables / functions | camelCase         | `currentPage`, `renderRuby()` |
| React components      | PascalCase        | `ShelfA`, `ViewerB`, `TweaksPanel` |
| Types / interfaces    | PascalCase        | `Story`, `TweakState` |
| Constants             | UPPER_SNAKE_CASE  | `TWEAK_DEFAULTS`, `FONT_PRESETS` |
| File names (component)| PascalCase.tsx    | `ShelfA.tsx`, `ViewerB.tsx` |
| File names (other)    | kebab-case.ts     | `ruby-parser.ts`, `tweaks-store.ts` |
| CSS classes           | kebab-case        | `eh-shelf`, `eh-btn`, `eh-viewer` |

### Code Style

- Indent: spaces, size 2
- Max line length: 100
- Comment language: 日本語 (子供向けプロダクトの語彙 / モックのコメントが日本語のため整合)
- Type annotations: Strict — 関数引数・戻り値・public API すべてに型注釈を付ける (`tsconfig.json` で `strict: true`)
- Recommended max lines per file: 300 行 (超える場合は分割を検討)
- 関数の責務は単一に保ち、副作用 (`useEffect`) は明示的にグループ化する
- 数値リテラル・カラー値・パディング等のマジックナンバーは CSS 変数か `const` に切り出す

### TypeScript / React 固有のルール

- `React.FC` は使用しない。関数コンポーネントの引数を直接型付け (`function ShelfA(props: ShelfAProps)`)
- props は interface ではなく `type` で定義 (拡張不要のケースが多いため)
- `any` は禁止。やむを得ない場合は `unknown` + 型ガード or `// TODO` コメントを付与
- `useState` の初期値が複雑なときは関数初期化形式 `useState(() => ...)` を使う
- グローバルスタイルは `:root` で CSS 変数として宣言。コンポーネント固有スタイルは CSS Modules
  か co-located `*.module.css` を推奨
- アクセシビリティ: ナビボタン・トグル等のインタラクティブ要素には必ず `aria-label` を付ける
- ふりがな表示は `<ruby>` / `<rt>` を用い、ON/OFF は CSS の `display` 制御で切り替える
  (DOM ツリーは保持しスクリーンリーダー互換性を維持)

## Git Rules

### Commit Messages (Conventional Commits)

```
{prefix}: {short summary} (TASK-{N})

- {bullet points of implementation details}
- Related UC: UC-XXX (該当時のみ)

Co-Authored-By: Claude <noreply@anthropic.com>
```

| prefix | Usage |
|--------|-------|
| `feat:` | 新機能 / 新ページ / 新コンポーネント |
| `fix:` | バグ修正 |
| `refactor:` | リファクタリング (動作不変) |
| `style:` | 見た目・CSS のみの変更 |
| `test:` | テストの追加・修正 |
| `docs:` | ドキュメントの追加・修正 |
| `chore:` | 設定 / 環境 / 依存更新 |
| `ci:` | CI/CD 関連 |

### Branch Strategy: GitHub Flow

- `main` を常時デプロイ可能な状態に保つ
- 機能/修正ごとに `feat/{slug}` または `fix/{slug}` ブランチを切る
- `main` への直接コミット禁止 (実装系 agent も同様)
- PR で main にマージ。PR 本文には `Closes #N` を記載する

### Staging

- `git add -A` は **禁止** (機微ファイル混入防止)
- 明示的に `git add {target-files}` でステージする
- `.env`, `credentials.*`, `*.secret`, `*.pem`, `*.key` は絶対にコミットしない

## Artifact Output Path

Aphelion 各エージェントが生成する成果物 Markdown は、**プロジェクトルート直下ではなく
`docs/` 配下** に配置する (本プロジェクト固有のルール)。

| 成果物 | 配置先 |
|--------|--------|
| `INTERVIEW_RESULT.md` / `RESEARCH_RESULT.md` / `POC_RESULT.md` / `CONCEPT_VALIDATION.md` / `SCOPE_PLAN.md` / `DISCOVERY_RESULT.md` | `docs/` |
| `SPEC.md` / `ARCHITECTURE.md` / `UI_SPEC.md` / `VISUAL_SPEC.md` / `TEST_PLAN.md` / `TASK.md` / `DELIVERY_RESULT.md` | `docs/` |
| `SECURITY_AUDIT.md` / `RELEASE_NOTES.md` / `OBSERVABILITY.md` / `OPS_PLAN.md` / `OPS_RESULT.md` / `MAINTENANCE_RESULT.md` | `docs/` |
| 設計メモ (Phase ごとの slug) | `docs/design-notes/<slug>.md` |
| Doc Flow 顧客向け成果物 (HLD / LLD / API reference 等) | `docs/deliverables/<slug>/*.md` |
| アーカイブ対象 (Closed Issue 連動) | `docs/design-notes/archived/<slug>.md` |

**例外** (ルート配置のまま):
- `README.md` — リポジトリ標準
- `CHANGELOG.md` — リポジトリ標準
- `CLAUDE.md` — Claude Code 規約

エージェントを起動する際、上記ハンドオフファイルが `docs/` にあることを前提に検索してください。
古いセッションでルート直下に生成された場合は `mv` で `docs/` に移動してから次のフェーズに進むこと。

## Project-Specific Rules

### コンテンツ・著作権

- 物語本文は著作権切れの古典童話の再話のみを格納する (赤ずきん, 桃太郎, 白雪姫, つるの恩返し,
  ブレーメンの音楽隊, かさじぞう など)
- LLM で新規生成する場合は以下の制約を守る:
  - 対象年齢: 3〜5 歳 (未就学児)
  - 文体: ひらがな主体, やさしい言葉, 敬体不要
  - 構成: 起承転結を持つ 5〜7 ページ
  - 出力に難しい漢字を含む場合は必ずルビ記法 `漢字{かんじ}` で表現する
- 挿絵は当面プレースホルダー (絵文字 + 色面)。実画像導入時は `pages[i].scene` をキーに管理

### アクセシビリティ (子供向けプロダクト固有)

- ふりがなは `<ruby>` を使用しスクリーンリーダー互換を保つ
- ナビゲーションはキーボードでも完結させる (`←` / `→` / `Esc`)
- 4.5:1 のコントラスト比を担保 (夜モードの `--mustard` は要検証)
- hover 効果は `@media (hover: none)` で抑制し、タッチデバイスを阻害しない
- フォントサイズは 16〜36px の範囲でユーザー調整可能にする

### 設定の永続化

- Tweaks (フォント, ふりがな, 文字サイズ, 夜モード, アクセント色, 本棚/ビュアーのバリアント)
  は localStorage に永続化する
- 本棚/ビュアーのバリアントは URL クエリでも切り替え可能にし、URL 共有時に同じ見た目を再現できる
  ようにする (任意目標)

### LLM 連携 (将来)

- API キー等の機微情報は `.env.local` に置き、コミット禁止 (Hook B が `.env*` への書き込みを
  ブロックする)
- 本番では Vite の `import.meta.env.VITE_*` でクライアント露出する変数のみを扱い、秘密キーは
  サーバ側 (将来の API ルート or BFF) で保持する
- LLM 呼び出しはフロントから直接ではなく、必ず自前のサーバエンドポイント経由にする (鍵漏洩
  防止)

### モック資産の扱い

- 既存の `Ehon.html` / `app.jsx` / `tweaks-panel.jsx` 等は **本番コードとして直接使わない**
- 本実装の参照用として `mock/` ディレクトリに退避し、必要に応じてデザイン確認に利用する
- `tweaks-panel.jsx` のホスト連携プロトコル (`__edit_mode_*` postMessage) は本実装では不要
